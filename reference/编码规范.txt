一、代码通用要求：
1，中间变量拆分、显式解引用
2，建一个utils写 let log = console.log.bind(console) 代替console.log
3，函数、变量、文件夹命名用驼峰命名发
4，所有函数必须拆分，高内聚，低耦合
5，不要使用三元表达式，不能简化语法，要显式控制流 / 显式条件块
6，任何代码如果因为声明提升才能正常执行，该代码视为不合格，必须重写
7，函数定义必须let 函数名 = function() {} 的方式
8，if else 嵌套不能超过两层，如果必须就用表驱动发, 不得使用switch case
9，变量的定义应该在离他最近的地方使用和定义
10，函数的调用在下 声明在上，保证代码从上往下执行
11，显式条件建模，条件预计算
12，禁止一行表达多个语义
13，所有函数必须明确同步或异步语义
14，代码缩进必须是4

二、前端react要求
    所有 React 组件必须使用函数组件，禁止使用 class component
    不同功能必须拆分为独立组件，禁止单组件承载多个业务职责
    UI 展示组件与业务逻辑组件必须分离
    组件文件必须只表达单一职责
    JSX 中禁止书写复杂逻辑，所有判断与计算必须提前在函数体中完成
    条件渲染必须使用显式条件变量，不得在 JSX 中进行多重判断
    所有事件处理函数必须抽离为具名函数，不得在 JSX 中定义匿名函数
    useEffect 必须职责单一，一个 useEffect 只处理一类副作用
    禁止在 useEffect 中同时处理请求、状态计算和副作用逻辑
    组件状态必须最小化，禁止保存可由现有状态计算得到的派生状态
    所有异步逻辑必须封装在独立函数中，不得直接写在组件渲染流程中
    组件之间通信必须通过 props 或受控回调完成，禁止隐式依赖
    禁止在组件中直接操作全局变量或全局状态
    组件命名必须清晰表达业务含义，禁止抽象或泛化命名
    组件目录结构必须反映功能划分，不得出现“通用大组件”目录

三、后端node.js要求
    后端项目必须严格采用 MVC 架构
    Controller 层只负责参数校验、请求分发和响应返回，不得包含业务逻辑
    Service 层负责完整业务逻辑处理，不得依赖 HTTP 请求或响应对象
    Model 层只负责数据结构定义与数据持久化，不得包含业务流程判断
    各层之间只允许单向调用，禁止跨层或反向依赖
    路由层只做路由与 Controller 映射，不得包含任何业务代码
    所有业务逻辑必须集中在 Service 层，禁止分散实现
    错误必须通过统一的错误机制处理，禁止在 Controller 层吞掉异常
    所有异步操作必须显式处理异常，不得依赖隐式错误传播
    禁止在 Model 层直接操作外部系统或全局状态
    数据校验必须在进入业务逻辑之前完成
    不同业务模块必须拆分为独立的 Controller、Service 和 Model
    文件与目录结构必须清晰反映业务域边界
    禁止在任意层中出现与其职责无关的逻辑
    所有对外依赖必须通过显式注入或参数传递，不得隐式引用
